## 技术栈确认（前端）

### 移动端框架
- **框架**：Vue 3 + TypeScript + Vite
- **跨平台**：Capacitor（Android）
- **UI组件库**：Vant 4
- **状态管理**：Pinia
- **样式**：UnoCSS

### 悬浮窗方案
- Capacitor Plugin：@capacitor/status-bar + 自定义悬浮窗
- 前端：Vue Composition API 实现悬浮窗逻辑

---

## 技术栈确认（后端）

### 框架与语言
- **框架**：FastAPI (Python)
- **数据库**：MySQL
- **缓存**：Redis
- **认证**：JWT

### API设计规范
- RESTful API
- 统一响应格式：{ code, message, data }
- JWT Token认证

## 数据表设计（初稿）

### 用户表 (users)
```sql
- id (PK)
- username
- avatar
- email
- vip_level
- created_at
```

### 宠物表 (pets)
```sql
- id (PK)
- name
- category
- image_url
- bloodline
- stats (JSON: HP/攻击/防御/魔攻/魔防/速度)
- recommended_nature
- recommended_talent
- recommended_skills
- capture_location
- capture_condition
- can_evolve
- evolution_forms (JSON)
- has_albinism
```

### 帖子表 (posts)
```sql
- id (PK)
- user_id (FK)
- title
- content
- images (JSON)
- team_id (FK, nullable) -- 阵容推荐关联
- board_id (FK)
- likes_count
- comments_count
- created_at
```

### 评论表 (comments)
```sql
- id (PK)
- post_id (FK)
- user_id (FK)
- parent_id (FK, nullable) -- 楼中楼
- content
- likes_count
- created_at
```

### 阵容表 (teams)
```sql
- id (PK)
- user_id (FK)
- name
- pets (JSON: [pet_id1, pet_id2, ...])
- description
```

### 世界频道表 (world_messages)
```sql
- id (PK)
- user_id (FK)
- content
- created_at
```
